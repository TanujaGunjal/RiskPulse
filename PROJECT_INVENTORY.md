## 🗄️ Database Schema

### Transaction Model (17 fields)
- **Identifiers**: caseid (unique), id
- **Demographics**: age, wealth_idx, education, residence
- **Medical History**: has_diabetes, has_htn
- **Scores**: DVS, TNS, ICS, CCS, HCFD_score, rule_score, anomaly_score, final_risk
- **Classification**: risk_tier
- **Audit**: explanation, created_at
- **Indexes**: caseid, risk_tier, created_at (for performance)

### Alert Model (5 fields)
- **Link**: transaction (ForeignKey, CASCADE)
- **Data**: risk_tier, final_risk, message
- **Status**: is_resolved (default False)
- **Audit**: created_at
- **Indexes**: is_resolved, created_at (for quick filtering)
- **Auto-Create**: When Transaction final_risk > 0.75

### AuditLog Model (5 fields)
- **Action**: action, user, endpoint
- **Security**: ip_address (GenericIPAddressField)
- **Audit**: timestamp
- **Indexes**: user+timestamp, action+timestamp (for compliance queries)
- **Auto-Log**: Every API request

## 🚀 Getting Started: Next Steps

### Immediate (Do This First)
1. **Run migrations**: `python manage.py makemigrations && python manage.py migrate`
2. **Create superuser**: `python manage.py createsuperuser`
3. **Load sample data**: `python manage.py seed_db --csv-path api/management/commands/sample_data.csv --clear`

### Then Start Servers
- **Backend**: `python manage.py runserver` (Terminal 1)
- **Frontend**: `cd frontend && npm install && npm start` (Terminal 2)

### Finally Test
- Visit http://localhost:3000 in browser
- Login with superuser credentials
- Test scoring and dashboard
