╚════════════════════════════════════════════════════════════════════════════╝

🎯 NFHS DATA NOT IN DATABASE
   ✓ Why: Reduces DB size, protects privacy
   ✓ Usage: Training ML models only (.pkl files stored locally)
   ✓ Storage: Only user-entered patient records kept

🎯 ML ENSEMBLE APPROACH
   ✓ 4 models weighted: XGBoost 35%, ISO Forest 20%, 
                         Autoencoder 15%, HCFD Rules 30%
   ✓ Fallback: Graceful degradation if models unavailable
   ✓ Output: Risk score + explanation + violations

🎯 POSTGRESQL FOR RUNTIME DATA
   ✓ Transactions: User-entered patient records
   ✓ Decisions: Insurance approval/denial tracking
   ✓ Audit: Complete action history
   ✓ Explainability: HCFD-XAI output storage
