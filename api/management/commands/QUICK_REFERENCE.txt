seed_db QUICK REFERENCE CARD

BASIC COMMANDS
--------------

# Load default CSV file
python manage.py seed_db

# Load custom CSV
python manage.py seed_db --csv-path /path/to/file.csv

# Load with progress (every 1000 rows)
python manage.py seed_db --csv-path data.csv

# Clear and reload
python manage.py seed_db --csv-path data.csv --clear

# Test with first 100 rows
python manage.py seed_db --limit 100

# Combine options
python manage.py seed_db --csv-path data.csv --clear --limit 1000

ERROR RECOVERY
--------------

# - Check file path is correct
# - Use absolute path: --csv-path /home/user/data.csv

# If "missing required columns"
# - Verify CSV has all required headers
# - Check column names (case-sensitive)
# - See sample_data.csv for format

# If partial load (some rows errored)
# - Check error messages in output
# - Fix CSV and retry with --csv-path
# - Idempotent: won't duplicate fixed rows

# If database locked (SQLite)

# - Try again

PERFORMANCE NOTES
-----------------

Small files (< 1K rows):    Few seconds
Medium files (10K rows):     30-60 seconds  
Large files (100K+ rows):    Several minutes

Use --limit for testing first on large files:
    python manage.py seed_db --csv-path huge.csv --limit 1000

For very large files, consider:
    1. Split into chunks: split -l 50000 large.csv chunk_
    2. Load chunks: python manage.py seed_db --csv-path chunk_aa
    3. Load incrementally (safe because idempotent)

  - caseid: text

  - has_diabetes, has_htn: boolean (true/false, 0/1, yes/no)
  - DVS, TNS, ICS, CCS, HCFD_score, rule_score, anomaly_score, final_risk: number (decimal, 0-1)

WHAT GETS CREATED
-----------------

For each CSV row:
- 1 Transaction object
- 1 Alert object (if final_risk > 0.75)
- 1 AuditLog entry (if audit logging enabled)

Example from sample_data.csv (15 rows):
- Transactions: 15
- Alerts: 10 (rows with final_risk > 0.75)

IDEMPOTENCY
-----------

The command is IDEMPOTENT - safe to run multiple times:
- Skips caseid values that already exist
- Won't create duplicate transactions
- Can run incrementally:
    python manage.py seed_db --csv-path batch1.csv
    python manage.py seed_db --csv-path batch2.csv

USE --clear FLAG IF:
- You want to replace ALL existing data
- First time setup with clean database
- Testing/development (non-production)

DON'T USE --clear IF:
- Adding to existing data (use without flag)
- Production database (data loss risk!)
- You need to preserve some records
